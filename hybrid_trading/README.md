# Hybrid Trend Following + Mean Reversion Trading System

A sophisticated algorithmic trading system that combines trend-following and mean reversion strategies for the Zerodha Kite platform.

## Overview

This system implements a hybrid approach to algorithmic trading:

- **Trend Engine**: Captures large directional moves with low-frequency entries based on price structure
- **Mean Reversion Engine**: Profits from short-term pullbacks and hedges trend positions during extended moves
- **Risk Management**: Comprehensive risk controls with automatic kill switch integration
- **Position Management**: Maintains logical separation between strategies while respecting Zerodha's net-position constraints

## Core Data Models

### Tick
Represents a single tick of market data with timestamp, symbol, price, and volume.

### Candle
OHLC candle for a specific timeframe with validation and utility properties (body_size, range_size, is_bullish, is_bearish).

### Signal
Trading signal generated by an engine (trend or mr) with type, quantity, reason, and price.

### Trade
Completed trade with timestamp, quantity, price, and trade type.

### MRTrade
Active mean reversion trade with impulse tracking and retracement calculation.

### OrderResult
Result of order execution with status, filled quantity, and average price.

## Enumerations

### TrendState
- UPTREND: Higher highs and higher lows
- DOWNTREND: Lower highs and lower lows
- NEUTRAL: No clear trend

### MRState
- EXTENDED_UP: Price extended above normal range
- EXTENDED_DOWN: Price extended below normal range
- NORMAL: Price within normal range

### SignalType
- ENTRY_LONG: Enter long position
- ENTRY_SHORT: Enter short position
- EXIT_PARTIAL: Partial position exit
- EXIT_FULL: Full position exit

## Configuration

The system uses a comprehensive configuration schema with validation:

- **SystemConfig**: Main configuration container
- **TrendConfig**: Trend engine parameters
- **MRConfig**: Mean reversion engine parameters
- **RiskConfig**: Risk management parameters
- **ExecutionConfig**: Order execution parameters

### Loading Configuration

```python
from hybrid_trading.config import SystemConfig

# Load from YAML file
config = SystemConfig.from_file('config.yaml')

# Validate configuration
config.validate()

# Access nested configurations
print(config.trend_config.base_position_size)
print(config.risk_config.max_daily_loss_pct)
```

### Creating Configuration Programmatically

```python
from hybrid_trading.config import SystemConfig, TrendConfig, MRConfig

config = SystemConfig(
    timeframes=['5m', '15m'],
    instruments=['NIFTY FUT'],
    execution_mode='automated',
    trend_config=TrendConfig(base_position_size=2),
    mr_config=MRConfig(max_mr_trades_per_leg=5)
)

config.validate()
```

## Usage Example

```python
from datetime import datetime
from hybrid_trading.data import Tick, Candle
from hybrid_trading.common import TrendState, SignalType
from hybrid_trading.execution import Signal, MRTrade
from hybrid_trading.config import SystemConfig

# Create a tick
tick = Tick(
    timestamp=datetime.now(),
    symbol='NIFTY FUT',
    last_price=18500.0,
    volume=100
)

# Create a candle
candle = Candle(
    timestamp=datetime.now(),
    open=18450.0,
    high=18520.0,
    low=18440.0,
    close=18500.0,
    volume=5000,
    timeframe='5m'
)

# Check candle properties
print(f"Body size: {candle.body_size}")
print(f"Is bullish: {candle.is_bullish}")

# Create a signal
signal = Signal(
    signal_type=SignalType.ENTRY_LONG,
    engine='trend',
    quantity=1,
    reason='Pullback to structure in uptrend',
    timestamp=datetime.now(),
    price=18500.0
)

# Create an MR trade
mr_trade = MRTrade(
    entry_time=datetime.now(),
    entry_price=18550.0,
    quantity=1,
    direction='short',
    impulse_start=18450.0,
    impulse_end=18600.0
)

# Calculate retracement
retracement = mr_trade.retracement_pct(18520.0)
print(f"Retracement: {retracement:.1f}%")

# Load configuration
config = SystemConfig.from_file('config.yaml')
print(f"Execution mode: {config.execution_mode}")
print(f"Max daily loss: {config.risk_config.max_daily_loss_pct}%")
```
